# Composition Workflow — 开头前 10 分钟

## When to use

用户带着一句模糊描述来了 —— "想写首伤感的钢琴"、"给短片配个悬疑段落"、"lofi 一段"。他们没给 BPM、没给调、没给和声。**不要反问**。这个 skill 教你如何在 5-10 分钟内把 4-8 小节的可听动机放进 REAPER，让用户"听到"而不是"想象"，然后基于听感迭代。已经有和声或旋律的用户不走这里 —— 那是 arrangement 的活。

## 1. 先出手：从关键词到默认参数（30 秒内决定）

用户情绪关键词 → 直接落表。**不要问，先写下来告诉用户"这是我的默认，30 秒不合适就改"**：

| 情绪词 | BPM | 调/调式 | 拍号 | 首选进行 |
|---|---|---|---|---|
| 伤感 / 安静 / 钢琴 | 68-76 | A minor (Aeolian) | 4/4 | i-VI-III-VII (Am-F-C-G) |
| 温暖 / 治愈 / 民谣 | 88-96 | G major | 4/4 | I-V-vi-IV (G-D-Em-C) |
| 悬疑 / 紧张 / 配乐 | 60-72 | D minor (Dorian 也可) | 4/4 或 6/8 | i-bII-i-V (Dm-Ebmaj-Dm-A) |
| Lofi / chill | 72-84 | F# minor | 4/4 | i-VII-VI-V (F#m-E-D-C#) 加 7th/9th |
| 战斗 / 史诗 | 130-150 | E minor (Phrygian 变体) | 4/4 | i-bII-i-bVII (Em-Fmaj-Em-Dmaj) |
| 欢快 / 流行 | 100-118 | C major 或 D major | 4/4 | I-V-vi-IV 或 vi-IV-I-V |

**术语速释**：
- **调式 mode** — 大调/小调之外的音阶变体。**Aeolian** = 自然小调（就是我们说的"小调"，暗）。**Dorian** = 小调把 b6 变成 6（明一点，Miles Davis 常用）。**Phrygian** = 小调把 2 变成 b2（更暗，西班牙/中东味）。
- **罗马数字** — 大写 = 大和弦（I、V），小写 = 小和弦（i、vi），b = 降半音（bII 就是主音上方半音的大和弦）。

## 2. 用 reaper-mcp 直接落 4 小节动机（不要先问）

上面的“模糊意图默认流程”只在用户没有给出精确施工范围时使用。用户或评测任务已经明确轨道、item、调性与交付范围时，必须严格按该范围施工：只要求 MIDI 就不要擅自添加 FX、试听音源、marker、渲染或改 tempo；“C 大调调内”不包含借用和弦或调外音，除非用户明确要求 modal interchange。额外的“让它更好听”不是越界写入的理由。

流程写死，以 A minor 68 BPM 为例：

1. `add_marker` 标 "motif v1 @ 68BPM Am"
2. `add_midi_notes` 在轨 1 写和声，每小节一个和弦（whole note，以四分音符为一拍、4 拍占满整小节）：
   - 小节 1: Am = A3 + C4 + E4
   - 小节 2: F  = F3 + A3 + C4
   - 小节 3: C  = C4 + E4 + G4
   - 小节 4: G  = G3 + B3 + D4
   （**根位三和弦** = 根音在最低的三音和弦，最稳最干净的堆叠）
3. `add_midi_notes` 在轨 2 写主旋律 8 个音，只用 A 小调音阶（A B C D E F G）：
   - 前 4 音上行到 **属音 = E5**（属音 = 音阶第 5 音，A 小调里就是 E）：例如 A4 - C5 - D5 - E5
   - 后 4 音回落到 **主音 = A4**（主音 = 音阶第 1 音）：例如 D5 - C5 - B4 - A4
   - 中间至少放一个 **跳进**（乐理上 3 度即算跳进，这里要 ≥ 4 度，比如 A4 → D5 是 4 度跳进）。全部相邻音阶叫 **级进**，听起来像练习曲。
   - 这个上行-下行轮廓就是 **tension→resolve**（紧张→释放）：耳朵走到属音时觉得"悬着"，回主音时觉得"到家"。
4. `render_to_wav`（不填路径，落 %TEMP%/prism-renders），把 wav 路径给用户，说："这是 v1，30 秒改一次。你要更暗/更快/更抒情？"

**关键点**：先给声音，再讨论。用户听 20 秒能说的比看谱聊 5 分钟多。

### 写后读回：最终说明只能来自工程事实

创建或改写 MIDI 后、向用户报告之前，必须完成一次自动读回；这是工具验证，不是人工审批：

1. 用 `get_midi_notes` 读取刚写入的 item，不要只复述计划中的音符数组。
2. 按 `start_beats` 分组，核对每小节的实际音高、数量和长度；和弦计划与工具参数必须使用同一个小节顺序。
3. 如果读回结果与计划不同，优先修正；若结果本身仍满足用户目标，则如实描述实际结果，不得把原计划写成已经落地。
4. 最终回复中的和弦名、顺序、BPM、轨道数和小节数只能来自读回状态。无法读回时明确说“尚未验证”，不要声称已经完成某个具体进行。

例如计划是 `C-G-Am-F`，就应核对 beat `0/4/8/12` 上的 pitch class 分别构成这四个和弦；不能实际写成 `Am-F-C-G`，最终文字却仍报告 `C-G-Am-F`。

## 3. 迭代节奏：一次只改一个变量（每步带工具调用）

用户反馈 → 一次改一件事，重渲染：

- **太快** → BPM -8（REAPER 主控 tempo 改，不动 MIDI），`render_to_wav` v2
- **太暗** → 换 **平行大调**（相同主音的大调，A minor → A major，把 C→C#、F→F#、G→G#）；用 `get_midi_notes` 读回旋律轨，改三个音的 pitch 后 `replace_midi_notes` 整体替换（`add_midi_notes` 是追加，会和旧音叠成双份）；`add_marker` v2
- **不够抓耳** → `get_midi_notes` 读主旋律，把第 2 小节强拍（beat 1）的音改成整段最高音（比如 E5 改成 G5），并把它前一个音改成低 5 度制造跳进；只动两个音可直接 `update_midi_note` 逐音改，改动多就 `replace_midi_notes`
- **太"套路"** → 把进行第 3 个和弦（C）替换为 **secondary dominant**（次属和弦 = 临时把下一个和弦当主音，为它写它自己的属和弦；C 的下一个是 G，G 的属和弦是 D7，所以把 C 换成 D7 = D3+F#3+A3+C4）；`get_midi_notes` 读回和声轨、改小节 3 的和弦音后 `replace_midi_notes`

每次改完 `add_marker` 打 v2/v3/v4，`render_to_wav` 让用户对比。**不要一次改三个参数，用户没法说清是哪个改动奏效**。

## 4. 何时闭嘴写 MIDI，何时开口问

**闭嘴写**：BPM 范围、默认调、首选进行、旋律轮廓 —— 全部按表默认。

**开口问**（且只在 v1 出声后问）：
- "要人声 demo 还是纯器乐？"（决定要不要留 200Hz-4kHz 频段空位）
- "长度目标？30 秒预告 / 2 分钟 A-B / 完整 3 分半？"（决定要不要写 B 段）
- "有参考曲吗？"（用户主动给才用；不给按默认走）

**不要问**："你喜欢什么风格？""想要什么感觉？" —— 用户已经在第一句说了。

## 5. Kontakt / 音色说明（诚实边界）

如果用户说"用某某钢琴音色" —— **明说：**Kontakt / Omnisphere 等重采样器的具体音色库必须在插件窗口内由用户手动加载**——NI 设计如此,MCP 只能 add_track_fx 挂上宿主,选不了具体 patch。
你能做的：`list_installed_fx` 查有没有 Kontakt，`add_track_fx` 挂到轨 1，然后告诉用户："Kontakt 已挂在轨 1，请你手动选 Alicia's Keys / Noire / 你想用的音色。"

## Anti-patterns

- **上来就问"你想要什么风格什么 BPM"** —— 用户不知道，所以才来找你。先给默认。
- **写完 32 小节才让用户听** —— 4 小节就 `render_to_wav`，别憋大招。
- **一次改 BPM+调+进行三样** —— 迭代信号丢失，用户耳朵分不清哪个起了作用。
- **纯级进旋律（8 个音全相邻音阶）** —— 听起来像哈农练习曲。至少一个 ≥4 度跳进。
- **和声用密集堆叠（四个音全在中央 C 附近一坨）** —— 根位三和弦、宽间距（根音在低八度，3+5 在中央 C 附近）比密堆好听十倍。
- **不打 marker 就迭代** —— v3 之后你和用户都不知道 v1 什么样，无法回滚。每版 `add_marker`。
- **默认调硬套 A minor 不看用户情绪词** —— 表是分档的，"欢快"用 A minor 会拧巴，按表落。
- **精确任务里擅自加试听 FX 或借用和弦** —— 已给出范围时只做范围内工程编辑；先满足可核验约束，再谈创意扩展。
